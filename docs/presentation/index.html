ODIM‑MH Presentation Demo

Minimal, factual demo used in the talk: raw files → import → observations → daily features → weekly decision.

Artifacts

  • demo.ttl — demo individuals (TTL)
  • Raw CSVs: steps.csv, sleep.csv (README)
  • Figure: Demo subgraph (simplified flow)
  • Queries:
    • demo_traceability.rq
    • demo_units.rq
    • demo_context_threshold.rq

Run Locally (inside tools container)

tooling/run_ontology_tools.sh build
# Merge ontology + demo for querying
tooling/run_ontology_tools.sh exec -- bash -lc \ 
  "robot merge --input mhm_ontology.owl --input docs/presentation/demo.ttl --output build/demo-merged.owl"

# Run queries (stdout)
tooling/run_ontology_tools.sh exec -- sparql --data build/demo-merged.owl --query docs/presentation/queries/demo_traceability.rq
tooling/run_ontology_tools.sh exec -- sparql --data build/demo-merged.owl --query docs/presentation/queries/demo_units.rq
tooling/run_ontology_tools.sh exec -- sparql --data build/demo-merged.owl --query docs/presentation/queries/demo_context_threshold.rq

All modeling uses the existing ODIM‑MH classes/properties (DL‑safe), with local SKOS context for “Travel”.
